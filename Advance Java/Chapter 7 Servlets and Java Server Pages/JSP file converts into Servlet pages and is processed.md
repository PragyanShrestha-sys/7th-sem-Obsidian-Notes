## What Happens Behind the Scenes

```
JSP File (hello.jsp)                    Generated Servlet (hello_jsp.java)
        │                                            │
        │ (Container translates)                     │
        ▼                                            ▼
┌─────────────────────┐                  ┌─────────────────────────────────┐
│ <html>              │                  │ out.write("<html>");            │
│ <body>              │    →    →    →    │ out.write("<body>");            │
│ <%= new Date() %>   │                  │ out.write(new Date().toString());│
│ </body>             │                  │ out.write("</body>");           │
│ </html>             │                  │ out.write("</html>");           │
└─────────────────────┘                  └─────────────────────────────────┘
```

## The Proof

### Your JSP Code:
```jsp
<!DOCTYPE html>
<html>
<head>
    <title>Hello JSP</title>
</head>
<body>
    <h1>Hello from JSP!</h1>
    <p>Current time: <%= new java.util.Date() %></p>
</body>
</html>
```

### The Servlet That JSP Becomes (Approximate):
```java
// This is AUTO-GENERATED by the web container
// You NEVER write this yourself!

import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class hello_jsp extends HttpServlet {
    
    protected void service(HttpServletRequest req, HttpServletResponse resp)
            throws ServletException, IOException {
        
        resp.setContentType("text/html");
        PrintWriter out = resp.getWriter();
        
        // All HTML becomes out.write() statements!
        out.write("<!DOCTYPE html>\n");
        out.write("<html>\n");
        out.write("<head>\n");
        out.write("    <title>Hello JSP</title>\n");
        out.write("</head>\n");
        out.write("<body>\n");
        out.write("    <h1>Hello from JSP!</h1>\n");
        out.write("    <p>Current time: ");
        
        // <%= %> becomes Java code
        out.print(new java.util.Date());
        
        out.write("</p>\n");
        out.write("</body>\n");
        out.write("</html>\n");
    }
}
```

## The Translation Rules

| JSP Element | Becomes in Servlet |
|-------------|-------------------|
| **Plain HTML** | `out.write("HTML here");` |
| **`<%= expression %>`** | `out.print(expression);` |
| **`<% code; %>`** | `code;` (direct Java) |
| **`<%! declaration %>`** | Becomes class-level member |

## Simple Example of Translation

```jsp
<!-- Original JSP -->
<html>
<body>
    <% int x = 10; %>
    Value is: <%= x %>
</body>
</html>
```

```java
// Generated Servlet (simplified)
out.write("<html>");
out.write("<body>");
int x = 10;
out.write("Value is: ");
out.print(x);
out.write("</body>");
out.write("</html>");
```

## Why This Matters

| Understanding | Implication |
|---------------|-------------|
| **JSP = Servlet** | Both do the same thing (generate HTML) |
| **JSP is easier for HTML** | Write HTML naturally, not in strings |
| **Servlet is better for logic** | Write Java naturally, not in tags |
| **First JSP request is slower** | Must translate to servlet first |

## The Golden Rule

> **JSP is NOT a separate technology - it's just a convenient way to write servlets. The container translates every JSP into a servlet before executing it. That's why JSP and Servlets work seamlessly together!**

**Memory trick:**
```
JSP (Human-friendly) → Container translates → Servlet (Machine-friendly)
                         (behind the scenes)

Write in JSP, Run as Servlet!
```
