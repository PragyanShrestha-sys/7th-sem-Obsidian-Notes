## What is Event Handling?

**Event Handling** is the mechanism that makes GUI programs respond to user actions.

```
User Action (Click, Type, Move) → Event is Created → Event Handler Runs → Program Responds

Example:
User clicks button → Click event occurs → Event handler runs → "Hello World" appears
```

**Without Event Handling:** GUI looks pretty but does nothing when you click buttons.
**With Event Handling:** GUI comes alive and responds to user interaction.

---

## Why is Event Handling Used?

| Problem | Solution |
|---------|----------|
| User clicks "Save" button | Event handler saves the file |
| User types in text field | Event handler updates the data |
| User selects a menu item | Event handler executes the command |
| User closes a window | Event handler saves before exiting |

**Simple analogy:** Event handling is like assigning a remote control button to actually change the TV channel instead of just sitting there.

---

## The Event Handling Model (Delegation Event Model)

Java uses the **Delegation Event Model** with 3 main players:

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   EVENT SOURCE  │────→│      EVENT      │────→│ EVENT LISTENER  │
│  (Button, etc.) │     │  (ActionEvent)  │     │ (Handles event) │
└─────────────────┘     └─────────────────┘     └─────────────────┘

1. Source: The component that was clicked (button, menu, text field)
2. Event: The object that carries information about what happened
3. Listener: The object that contains the code to respond
```

## 1. Event Handling Concept

### How It Works

```java
// STEP 1: Create the event source (button)
JButton button = new JButton("Click Me");

// STEP 2: Create the event listener (handler)
ActionListener listener = new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("Button was clicked!");
    }
};

// STEP 3: Register the listener with the source
button.addActionListener(listener);
```

### Flow Diagram

```
User clicks button
       ↓
Button creates ActionEvent object
       ↓
Event is sent to registered listeners
       ↓
Listener's actionPerformed() method runs
       ↓
Your code executes (message prints, file saves, etc.)
```

### Common Event Types

| Event Class | When it occurs | Listener Interface |
|-------------|----------------|-------------------|
| `ActionEvent` | Button click, menu selection, Enter in text field | `ActionListener` |
| `MouseEvent` | Mouse click, press, release, enter, exit | `MouseListener` |
| `KeyEvent` | Key press, release, type | `KeyListener` |
| `ItemEvent` | Checkbox/radio button selection change | `ItemListener` |
| `FocusEvent` | Component gains/loses focus | `FocusListener` |
| `WindowEvent` | Window open, close, iconify, activate | `WindowListener` |

## 2. [[Listener Interfaces]]

**What is a Listener Interface?** An interface that declares methods for handling specific events.

## 3. [[Using Action Commands]]

**What is an Action Command?** A string that identifies the action, useful when multiple components share the same listener.

| Aspect | Description |
|--------|-------------|
| **Purpose** | Identify which component triggered the event |
| **Default** | Button's text (if not set) |
| **Use case** | Multiple buttons with one listener |

---
## 4. [[Adapter Classes]]

**What are Adapter Classes?** Classes that provide empty implementations of listener interfaces.
note: **Adapter classes are ONLY used with Listener interfaces, and they are pre-defined by Java specifically for those interfaces.**

## The Golden Rule

> **Event handling connects user actions to program responses. Sources generate events, listeners handle them. Use action commands when multiple components share one listener. Use adapter classes when you only need a few methods from a multi-method listener interface.**

**Memory trick:**
```
Source → Event → Listener → Response

Action Command = "I identify which button was clicked"
Adapter Class = "I let you override only the methods you need"
```
