**Data types are classified by their degree of organization: structured (rigidly organized), unstructured (no predefined structure), and semi-structured (hybrid). Structured data uses rows/columns, unstructured comprises files/media, and semi-structured uses tags (JSON/XML) to provide partial structure. The key difference lies in flexibility vs. ease of analysis.**

**Summary Table of Differences**

| Feature     | Structured         | Semi-Structured          | Unstructured          |
| ----------- | ------------------ | ------------------------ | --------------------- |
| Schema      | Predefined         | Flexible/Self-describing | No Schema             |
| Format      | Tables (Rows/Cols) | Tags/Markers (JSON, XML) | Files/Binary/Text     |
| Storage     | RDBMS (SQL)        | NoSQL/Data Lake          | Data Lake/File System |
| Flexibility | Low                | Medium                   | High                  |
| Querying    | Easy (SQL)         | Moderate                 | Difficult (NLP/AI)    |
| Example     | Customer Table     | JSON File                | Video File            |

**Key Usage Examples**

- **Structured:** Financial transactions, inventory records.
- **Unstructured:** CCTV footage analytics, social media monitoring, document management.
- **Semi-Structured:** E-commerce data feeds, configuration files, web scraping.
