

### What is a Slip Chart?


![[Pasted image 20260330144805.png]]


A **Slip Chart** (also known as a **Schedule Slippage Chart**) is a visual project management tool used to track and report schedule delays. In software project management, where requirements often change and estimation is difficult, the Slip Chart serves as an early warning system to detect when a project is falling behind its planned timeline.

As noted in your text, it plots two key dates against each other:
1.  **Announcement Date** (or Current Date/Review Date): The point in time when the status is being checked.
2.  **Planned Ship Date** (or Completion Date): The date the activity or project is scheduled to be finished.

### The "Gantt Chart" Variation

Your document describes a specific variation of the Slip Chart that functions similarly to a Gantt Chart:
- A **vertical line** represents the "status date" (today).
- **Activities to the left** of this line are completed.
- **Activities to the right** are not yet completed or are overdue.

In the example provided in your text (with modules Kelvin, Paul, Peter, and Zobel), if the vertical line is placed at week 15, a module like "Kelvin module F" showing completion dates stretching into week 20 indicates significant future work, while modules with bars ending before week 15 would be considered complete.

### Purpose in Monitoring and Control

In the "Monitoring and Control" phase of software project management, the Slip Chart serves three critical functions:

1.  **Visualizing Slippage:** It provides an immediate, intuitive visual indicator of trouble. Project managers do not need to read complex tables to see if the "ship date" is moving further to the right as the current date advances.
2.  **Tracking Accuracy:** It helps assess the accuracy of the original estimates. If most points lie below the line (to the right), it indicates a systematic underestimation of effort.
3.  **Facilitating Corrective Action:** By identifying *which* modules or activities are slipping (e.g., "Paul module C" or "Zobel module A" in your example), the manager can focus resources on the critical path items that are causing the overall project delay.

### Summary
A Slip Chart is essential for **Schedule Control**. Unlike a standard Gantt chart, which shows what should be done when, a Slip Chart explicitly highlights the gap between *planned* performance and *actual* performance, making it the definitive tool for managing deadlines in software projects.
