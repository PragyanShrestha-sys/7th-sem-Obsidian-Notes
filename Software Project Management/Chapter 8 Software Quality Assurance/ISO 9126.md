## The Six Quality Characteristics of ISO 9126

ISO 9126 defines software quality using **six main characteristics**, each with sub-attributes:

| Characteristic | Meaning | What It Measures |
|----------------|---------|------------------|
| **1. Functionality** | Does the software do what it should? | Correctness, security, interoperability, compliance |
| **2. Reliability** | Does it work without failure? | Maturity, fault tolerance, recoverability |
| **3. Usability** | Is it easy to use? | Learnability, understandability, operability |
| **4. Efficiency** | Does it use resources well? | Time behavior (speed), resource utilization |
| **5. Maintainability** | Is it easy to change? | Stability, analyzability, changeability, testability |
| **6. Portability** | Can it run on different platforms? | Adaptability, installability, coexistence |

## Summary

| Aspect | Detail |
|--------|--------|
| **Full Name** | ISO/IEC 9126 |
| **Title** | Software Engineering – Product Quality |
| **Purpose** | Standard framework for evaluating software quality |
| **Replaced by** | ISO 25000 (SQuaRE) |
| **Core Idea** | Quality = 6 characteristics + sub-attributes + metrics |

> **Takeaway:** ISO 9126 gives you a structured way to think about software quality. Instead of saying "good quality," you can say "high functionality, reliable, usable, efficient, maintainable, and portable."
