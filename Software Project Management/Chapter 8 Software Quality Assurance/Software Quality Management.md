## What is Software Quality Management (SQM)?

**Software Quality Management (SQM)** is a **management process** aimed at **developing and managing** the quality of software to ensure the product meets customer expectations, regulatory standards, and developer requirements.

Software quality managers use a **cyclical process-based** quality assessment to test software before release and identify and fix bugs.

---

## Components of SQM

The image identifies three main components:

| Component | Description |
|-----------|-------------|
| **Quality Assurance** | Activities to ensure that the software development **process and products meet specified requirements**. (Focuses on preventing defects) |
| **Quality Planning** | Identifying which **quality standards** are relevant to the project and determining how to satisfy them. |
| **Quality Control** | Monitoring and recording results to ensure that performance meets the required standards. (Focuses on detecting defects) |

## [[Software Quality Metrics]]

Software quality is measured using various metrics that evaluate different aspects of the software product. The image lists the following **10 quality metrics**:

## Summary Table

| Metric | Focus | Key Question |
|--------|-------|--------------|
| **Correctness** | Requirements | Does it do what it should? |
| **Reliability** | Failures | How often does it fail? |
| **Efficiency** | Resources | Is it fast and lean? |
| **Integrity** | Security | Is it secure? |
| **Usability** | User effort | Is it easy to use? |
| **Maintainability** | Change effort | Is it easy to fix/modify? |
| **Testability** | Test effort | Is it easy to test? |
| **Flexibility** | Adaptation | Can it adapt to new needs? |
| **Portability** | Platform change | Can it run elsewhere? |
| **Interoperability** | System coupling | Does it work with others? |

## Practical Application

| Stakeholder | Most Important Metrics |
|-------------|----------------------|
| **End User** | Correctness, Reliability, Usability, Efficiency |
| **Project Manager** | Correctness, Reliability, Maintainability |
| **Developer** | Maintainability, Testability, Flexibility |
| **Security Team** | Integrity |
| **System Architect** | Portability, Interoperability, Efficiency |

> **Takeaway:** No single metric defines software quality. A balanced approach using **multiple metrics** provides a complete picture. For example, software can be correct and reliable but unusable, or secure but inefficient. Measure what matters most to your stakeholders and project goals.

## What is ISO 9126?

**ISO 9126** is an international standard for **evaluating software quality**. It provides a framework for defining, measuring, and assessing the quality of software products.
