## What is Critical Path

The **critical path** is ==the longest continuous sequence of dependent tasks in a project==. It determines the absolute minimum time required to complete the entire project. Any delay to a task on this path will directly delay the project's final completion date.

Key Characteristics

- **Longest Timeline:** It represents the longest path of planned activities from start to finish.
- **Zero Float (Slack):** Tasks on the critical path have no buffer time. If they take longer than planned, the project end date shifts.
- **Dynamic:** The critical path can change as a project progresses, especially if unexpected delays cause non-critical paths to become the new longest sequence of tasks

A Simple Analogy: Packing for a Trip

Imagine you and a friend are getting ready to drive to the airport. You have two final tasks that you do at the same time:

- **Task A (You):** Putting on shoes takes **2 minutes**.
- **Task B (Friend):** Finding a lost passport takes **20 minutes**.

The **longest path** here is finding the passport (20 minutes).

Because you cannot leave without the passport, the **shortest possible time** before you can walk out the door is **20 minutes**. You cannot magically leave in 2 minutes just because your shoes are on.

## What Is CPM?

**CPM (Critical Path Method)** is a network planning technique used to identify the sequence of critical activities that determine the minimum project duration.  Unlike PERT which uses three time estimates, CPM uses a **single, deterministic time estimate** for each activity and focuses on identifying the **critical path** and **schedule flexibility**

It was developed in the late 1950s by DuPont and Remington Rand for industrial and construction projects.

## Key Concepts in CPM

| Concept                 | Explanation                                                                             |
| ----------------------- | --------------------------------------------------------------------------------------- |
| **Critical Path**       | The longest path through the network; determines the shortest possible project duration |
| **Critical Activities** | Activities on the critical path; any delay in these will delay the entire project       |
| **Float (Slack)**       | The amount of time an activity can be delayed without affecting the project end date    |
| **Early Start (ES)**    | The earliest time an activity can begin                                                 |
| **Early Finish (EF)**   | The earliest time an activity can finish (ES + Duration)                                |
| **Late Start (LS)**     | The latest time an activity can begin without delaying the project                      |
| **Late Finish (LF)**    | The latest time an activity can finish without delaying the projet                      |
| Critical event          | zero slack time                                                                         |
| Dependencies            | Relationships between activities                                                        |
| Duration                | Total time required to complete the Critical path activities                            |
|                         |                                                                                         |
