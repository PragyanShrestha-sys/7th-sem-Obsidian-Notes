A **network planning model** is a graphical technique used to represent, analyze, and manage the sequence of activities in a project. It shows **logical dependencies** between tasks and helps determine the **project duration**, **critical path**, and **schedule flexibility**.

Unlike a Gantt chart which focuses on time, a network model focuses on **relationships**—showing which tasks must come before others, which can happen in parallel, and where delays will impact the project.

## Common Types of Network Models

| Model        | Full Name                               | Description                                                                                 |
| ------------ | --------------------------------------- | ------------------------------------------------------------------------------------------- |
| **[[PERT]]** | Program Evaluation and Review Technique | Uses three time estimates (optimistic, most likely, pessimistic) to account for uncertainty |
| **[[CPM (Critical Patth method)]]**  | Critical Path Method                    | Uses single time estimates and focuses on identifying the critical path                     |
| **PDM**      | Precedence Diagramming Method           | Uses nodes for activities and arrows for relationships; supports multiple dependency types  |
