### Configuration Management Responsibilities

**Strategic – Setting Standards**  
- Define branching, naming, and change submission rules.  
- Choose tools and design system architecture.  
- Lock official release baselines.  
- Set repository access policies (read/write/admin).

**Tactical – How Daily Work Is Carried Out**  
- Add/remove developer access.  
- Trigger and monitor builds; flag broken commits.  
- Resolve complex merge conflicts.  
- Package releases for Operations.  
- Run daily backups and test restoration.

**Change Control – Controlling How the Project Changes Over Time**  
- Lead Change Control Board (CCB) meetings.  
- Assess impact of proposed changes on scope and schedule.  
- Enforce review and testing requirements on pull requests.  
- Manage hotfixes and ensure they merge back to main.

**Audit & Verification – Checking Progress Periodically**  
- Confirm the release meets requirements (functional audit).  
- Verify builds come from the correct source code (physical audit).  
- Maintain version and change history for leadership.  
- Produce compliance documentation for external auditors.

---

**Bottom Line:**  
> Keep the software in a known, reproducible, and stable state throughout its entire lifecycle.
