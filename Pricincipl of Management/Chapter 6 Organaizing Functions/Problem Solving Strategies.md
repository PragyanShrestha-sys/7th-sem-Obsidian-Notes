
### Strategy 1: Algorithm

**Definition:** A **step-by-step procedure** that, if followed correctly, will **always** produce a solution.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Systematic, guaranteed, exact |
| **Time** | Can be time-consuming |
| **Best for** | Well-structured, routine problems |
| **Example** | Following a recipe, a mathematical formula, or a troubleshooting guide |

**Example:** Troubleshooting a printer.

| Step | Action |
| :--- | :--- |
| 1 | Check if printer is plugged in. |
| 2 | Check if paper is loaded. |
| 3 | Check if ink/toner is present. |
| 4 | Check if connected to computer. |
| 5 | Restart printer. |

**Advantage:** Guaranteed to work if problem is simple and steps are correct.
**Disadvantage:** Not feasible for complex, ill-structured problems.

### Strategy 2: Heuristic

**Definition:** A **mental shortcut** or "rule of thumb" based on experience that **usually** works, but does not guarantee a solution.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Quick, practical, experience-based |
| **Time** | Fast |
| **Best for** | Routine problems, time pressure |
| **Example** | "If it's not broken, don't fix it." "When in doubt, ask your supervisor." |

**Common Heuristics in Management:**

| Heuristic | Meaning |
| :--- | :--- |
| **Satisfice** | Choose the first "good enough" solution. |
| **Elimination by aspects** | Eliminate alternatives that fail key criteria. |
| **Availability heuristic** | Judge likelihood based on how easily examples come to mind. |

**Advantage:** Fast, saves mental energy.
**Disadvantage:** Can be wrong; biases may lead to poor decisions.

### Strategy 3: Trial and Error

**Definition:** Trying different solutions **one by one** until one works. Learning from failures.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Experimental, learning-oriented |
| **Time** | Can be slow |
| **Best for** | Simple problems, low-risk situations |
| **Example** | Trying different keys to open a lock; trying different passwords. |

**Example:** A new manager trying to motivate a team.

| Attempt | Action | Result |
| :--- | :--- | :--- |
| 1 | Offer monetary bonus | No improvement |
| 2 | Give public recognition | Slight improvement |
| 3 | Offer flexible hours | Significant improvement → Solution found. |

**Advantage:** Works when no other method is available.
**Disadvantage:** Can be inefficient; may fail before finding solution.

### Strategy 4: Brainstorming

**Definition:** A **group creativity technique** where participants generate as many ideas as possible without criticism.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Creative, group-based |
| **Time** | Moderate |
| **Best for** | Ill-structured problems, need for innovation |
| **Example** | "How can we increase customer satisfaction?" |

**Rules of Brainstorming:**

| Rule | Explanation |
| :--- | :--- |
| No criticism | All ideas are welcome. No negative comments. |
| Go for quantity | More ideas = better chance of good ideas. |
| Encourage wild ideas | Unusual ideas may spark better ones. |
| Build on others' ideas | "Hitchhiking" or "piggybacking" is encouraged. |

**Advantage:** Generates many creative ideas.
**Disadvantage:** Can be dominated by loud voices; may produce many unusable ideas.

### Strategy 5: Root Cause Analysis (RCA)

**Definition:** A strategy that focuses on finding the **underlying cause** of a problem, not just treating the **symptoms**.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Analytical, deep-dive |
| **Time** | Can be time-consuming |
| **Best for** | Recurring problems, complex problems |
| **Tool example** | 5 Whys, Fishbone Diagram |

**Example: 5 Whys Method**

| Question | Answer |
| :--- | :--- |
| Why #1 | Why did the machine stop? → Fuse blew. |
| Why #2 | Why did the fuse blow? → Circuit overloaded. |
| Why #3 | Why did the circuit overload? → Bearing seized. |
| Why #4 | Why did the bearing seize? → No lubrication. |
| Why #5 | Why was there no lubrication? → Maintenance schedule not followed. |

**Root Cause:** Poor maintenance system.
**Solution:** Fix maintenance schedule (not just replace fuse).

**Advantage:** Prevents problem from recurring.
**Disadvantage:** Can be time-consuming; may oversimplify complex problems.

### Strategy 6: Divide and Conquer

**Definition:** Breaking a **large, complex problem** into **smaller, manageable sub-problems** and solving each individually.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Analytical, systematic |
| **Time** | Moderate to high |
| **Best for** | Large, complex problems |
| **Example** | Launching a new product (break into marketing, production, finance, HR sub-problems). |

**Example: "Increase company profitability"**

| Sub-problem | Focus |
| :--- | :--- |
| Sub-problem 1 | Reduce production costs |
| Sub-problem 2 | Increase sales revenue |
| Sub-problem 3 | Reduce administrative overhead |
| Sub-problem 4 | Optimize pricing strategy |

**Advantage:** Makes large problems manageable.
**Disadvantage:** Sub-problems may interact in unexpected ways.

### Strategy 7: Means-Ends Analysis

**Definition:** A strategy that continuously compares the **current state** to the **goal state** and takes actions that **reduce the difference**.

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Goal-oriented, step-by-step |
| **Time** | Moderate |
| **Best for** | Planning, goal-oriented problems |
| **Example** | GPS navigation; project planning |

**Example: Means-Ends Analysis for Career Goal**

| Current State | Goal State | Gap | Action to Reduce Gap |
| :--- | :--- | :--- | :--- |
| No management degree | MBA degree | 2 years study | Apply to university |
| No work experience | 5 years experience | 5 years | Get internship |
| Weak network | Strong professional network | 100 connections | Attend industry events |

**Advantage:** Clear, goal-focused.
**Disadvantage:** May miss creative alternative paths.

---

### Strategy 8: Analogical Reasoning

**Definition:** Using **solutions from similar past problems** to solve a current problem. "What worked before?"

| Aspect | Explanation |
| :--- | :--- |
| **Nature** | Experience-based, transfer learning |
| **Time** | Fast |
| **Best for** | Familiar problems in new contexts |
| **Example** | "We solved a supply chain problem in 2020 by finding a backup supplier. Let's try that again." |

**Advantage:** Fast, uses existing knowledge.
**Disadvantage:** Analogies may be imperfect; what worked before may not work now.
